# Демонстрация экрана выбора десертов — UI прототип

Этот репозиторий содержит только фронтенд‑прототип (интерфейс) хладомата — экран выбора десертов. По ссылке ТЗ, однако в коде реализована только визуальная часть экрана (интерфейс): визуальная сетка, выбор и корзина. Аппаратная логика, оплата, интеграция с контроллерами и датчиками — не реализованы.

Ссылка на исходное ТЗ (для справки):
- https://docs.google.com/document/d/1LMQWwAv7wg6SKJsGcvT3hchJVEOcYWQkpaEucDHwZYY

Кратко — что реально реализовано в коде
- Сетка товара 8×5 (40 ячеек) с данными из `src/data/desserts.json`.
- Отображение изображения товара и цены, бейдж скидки (если есть).
- Выбор/снятие выбора кликом по ячейке (визуальная индикация — фон салатного цвета при выборе).
- Правая панель (корзина) с карточками выбранных позиций: мини‑превью, описание, цена и аккуратная кнопка удаления.
- Адаптивный расчёт размеров ячеек: `App.tsx` вычисляет точный `cellSize` и передаёт в `DessertGridItem`.

Что НЕ реализовано (из ТЗ)
- Аппаратная часть: управление дверцами, охлаждением, датчиками наличия товара.
- Реальные LED‑ценники с аппаратной интеграцией.
- Процесс оплаты/эквайринг (нет backend и платежного шлюза).
- Открытие ячеек и выдача товара после оплаты.
- Логирование, отчёты, учёт остатков и интеграция с базой данных.

Важно: визуальная подсветка ячеек в интерфейсе (салатный фон для выбранных/отмеченных) — это чисто UI‑элемент для демонстрации и не управляет реальным оборудованием.

Как запустить локально

```bash
npm install
npm run dev
# открыть URL, который покажет Vite (обычно http://localhost:5173)
```

Ключевые файлы
- `src/App.tsx` — главный компонент: расчёт размеров сетки, рендер 8×5 и панель корзины
- `src/components/DessertGridItem.tsx` — компонент ячейки
- `src/data/desserts.json` — набор из 40 позиций (данные и описания)
- `public/desserts/` — изображения
